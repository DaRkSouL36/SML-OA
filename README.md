## ‚öôÔ∏è HOW IT WORKS

1. **TASK GENERATION**
   - SYNTHETIC IOT TASKS GENERATED USING LOGNORMAL AND GAMMA DISTRIBUTIONS.

2. **EXECUTION TIME PREDICTION**
   - LINEAR REGRESSION MODEL ESTIMATES EXECUTION LATENCY (R¬≤ ‚âà 0.97).

3. **COS COMPUTATION**
   - NORMALIZED TASK FEATURES ARE COMBINED USING WEIGHTED COS FORMULATION.

4. **STATISTICAL TIERING**
   - IQR-BASED THRESHOLDS ASSIGN TASKS TO IOT, EDGE, OR CLOUD.

5. **ANOMALY & STRESS HANDLING**
   - SKEWNESS, KURTOSIS, AND QUEUE STATES TRIGGER FALLBACK DECISIONS.

6. **LOGGING & VISUALIZATION**
   - ALL DECISIONS ARE LOGGED FOR POST-HOC ANALYSIS AND VISUAL DIAGNOSTICS.

## üîÑ OFFLOADING DECISION FLOW (SML-OA PIPELINE)

<p align="center">
  <img src="VISUALS\FLOWCHART.png" width="900"/>
</p>

THE FLOWCHART ILLUSTRATES THE END-TO-END CONTROL FLOW OF THE SML-OA PIPELINE:

- TASKS ARRIVE FROM IOT DEVICES INTO A SLIDING-WINDOW TASK QUEUE  
- FEATURES (TASK SIZE, CPU DEMAND) ARE EXTRACTED  
- EXECUTION TIME IS PREDICTED USING A LIGHTWEIGHT REGRESSION MODEL  
- FEATURES ARE NORMALIZED AND COMBINED INTO A COMPOSITE OFFLOADING SCORE (COS)  
- STATISTICAL METRICS (Q1, Q3, IQR, SKEWNESS, KURTOSIS) ARE COMPUTED  
- ANOMALIES ARE ISOLATED AND SANDBOXED WHEN REQUIRED  
- NORMAL TASKS FOLLOW STATISTICAL TIERING (IOT / EDGE / CLOUD)  
- SYSTEM STRESS TRIGGERS A FALLBACK ENGINE TO PRESERVE QOS  

THIS PIPELINE ENSURES DISTRIBUTION-AWARE, INTERPRETABLE, AND STRESS-RESILIENT TASK OFFLOADING.
